# Home / Feed / Portfolio Wireframes

기준 문서:

- `docs/frontend-blueprint.md`
- 현재 구현된 `HomeScreen`, `FeedScreen`, `PortfolioScreen`

목표:

- `Home`, `Feed`, `Portfolio`의 역할이 서로 겹치지 않게 분리한다.
- 모두 같은 카드 스택 구조를 쓰더라도 화면 성격은 분명하게 다르게 만든다.
- 모바일 우선으로 설계하되, 넓은 화면에서는 2열 확장 배치를 허용한다.

## 역할 분리 원칙

### Home

- 오늘 바로 확인해야 하는 숫자와 행동만 보여주는 `대시보드`
- 장기 분석보다 `오늘 상태`, `다음 행동`, `짧은 미리보기`에 집중

### Feed

- 친구 거래 흐름을 빠르게 훑고 반응하는 `타임라인`
- 상단 설명 카드보다 `콘텐츠 흐름`이 먼저 보여야 함

### Portfolio

- 내 자산을 깊게 읽는 `분석 화면`
- 보유 종목, 구성, 성과, 루틴 시각화를 가장 많이 담는 화면

## 공통 레이아웃 규칙

- 상단 타이틀 아래 첫 블록은 각 화면의 핵심 역할을 가장 잘 설명하는 `대표 블록` 1개만 둔다.
- 한 화면에서 같은 CTA를 2번 이상 반복하지 않는다.
- `추가`, `가져오기`, `거래 기록` 계열 CTA는 가능한 한 하단 `+` 탭으로 모은다.
- `Home`은 미리보기만 두고 상세는 `Feed`, `Portfolio`, `Profile`로 넘긴다.
- `Feed`는 상단 유틸보다 타임라인 본문이 먼저 시작되어야 한다.
- `Portfolio`는 유일하게 깊은 분석 블록과 보유 종목 목록을 함께 가져가도 된다.
- 빈 상태, 로딩 상태, 오류 상태도 각 화면 성격에 맞게 다르게 보이도록 구성한다.

## Home Wireframe

### 화면 목적

- 사용자가 앱을 열었을 때 `오늘 내 상태가 어떤지`와 `지금 무엇을 할지`를 5초 안에 파악하게 한다.

### 핵심 질문

- 오늘 내 자산은 올랐는가 내렸는가
- 오늘 확인하거나 기록해야 할 일이 있는가
- 친구 쪽에서 놓치면 안 되는 새 움직임이 있는가

### 모바일 와이어 구조

```text
[Header]
Today
오늘의 투자 현황

[Hero Summary Card]
총 평가금액
오늘 등락 / 총 수익률 / 보유 종목
짧은 보조 문장: 리마인더 또는 최근 동기화 상태

[Today Action Strip]
오늘 할 일 1개 강조
- 리마인더 확인
- 첫 포트폴리오 만들기
- 거래 기록 추가

[Quick Actions]
알림
리마인더
사람 찾기

[Portfolio Snapshot]
상위 보유 종목 3~4개
간단한 자산 구성 바 또는 미니 도넛
버튼: 포트폴리오 전체 보기

[Friend Activity Preview]
친구 거래 2건 미리보기
버튼: 피드 전체 보기

[Routine Preview]
리마인더 1~2개 또는 다음 투자 루틴
버튼: 리마인더 관리
```

### 섹션별 의도

#### 1. Hero Summary Card

- `Home`의 중심 블록
- 숫자 우선
- 현재 `Home`과 `Portfolio`가 겹치는 이유는 자산 구성 정보가 너무 크게 들어가 있기 때문
- 여기서는 깊은 분석 대신 `오늘 상태 요약`까지만 허용

#### 2. Today Action Strip

- 이 화면을 대시보드처럼 느끼게 만드는 핵심 블록
- 항상 1개의 우선 행동만 강조
- 상태에 따라 문구가 바뀌어야 함

예시:

- 등록된 포트폴리오 없음: `포트폴리오 상태 확인`
- 리마인더 예정일 도래: `오늘 적립식 투자 확인`
- 최근 거래 없음: `포트폴리오 / 피드 흐름 점검`

#### 3. Quick Actions

- 현재 여러 화면에 흩어진 반복 CTA를 `Home` 한 곳에 모으는 역할
- 2개 또는 3개까지만 유지
- 단, `추가 / 가져오기`는 하단 `+` 탭으로 보내고 여기서는 조회/관리 성격만 남긴다

#### 4. Portfolio Snapshot

- `Portfolio`로 보내기 위한 미리보기
- 상위 종목, 간단한 비중, 총평 정도만 노출
- 예상 배당, 섹터 상세, 전체 종목 리스트는 넣지 않음

#### 5. Friend Activity Preview

- 소셜 앱이라는 정체성을 잃지 않게 만드는 블록
- 길게 늘어놓지 말고 2건 미리보기만 유지

### 비워야 할 것

- 긴 리마인더 목록
- 내 최근 거래의 긴 목록
- 상세 섹터 분석
- 배당 월별 차트

### 넓은 화면 확장안

```text
좌측 2/3
- Hero Summary Card
- Portfolio Snapshot
- Friend Activity Preview

우측 1/3
- Today Action Strip
- Quick Actions
- Routine Preview
```

### Home 화면 톤

- 가장 빠르고 간결해야 함
- 화면 전체를 읽게 하지 말고, 위에서 아래로 바로 행동이 이어져야 함

## Feed Wireframe

### 화면 목적

- 친구들의 거래 흐름을 `스크롤 중심`으로 소비하게 한다.

### 핵심 질문

- 누가 어떤 거래를 했는가
- 내가 반응하거나 눌러볼 만한 거래가 무엇인가
- 새로운 사람을 찾거나 필터링해야 하는가

### 모바일 와이어 구조

```text
[Header]
Feed
친구 거래 타임라인

[Search / Filter Bar]
검색 입력 또는 빠른 필터 칩
전체 / 매수 / 매도 / 국내 / 미국 / 팔로우 중

[Inline Utility Row]
사람 찾기
새로고침

[Pinned Highlight Card]
지금 가장 눈에 띄는 거래 1건 또는 오늘 가장 많이 반응 받은 거래

[Timeline List]
거래 카드 반복
- 작성자
- 거래 요약
- 코멘트
- 리액션 / 댓글

[Load More / Empty / Follow CTA]
피드가 비었을 때 사람 찾기 CTA
```

### 섹션별 의도

#### 1. Search / Filter Bar

- 현재 설계 문서에는 들어가 있지만 실제 화면에서는 약한 부분
- `Feed`를 단순 리스트가 아니라 탐색 가능한 화면으로 만들어 줌
- 상단에 큰 설명 카드 대신 이 줄이 먼저 와야 함

추천 필터:

- 전체
- 매수
- 매도
- 국내
- 미국

#### 2. Inline Utility Row

- 버튼 2개 이내
- `사람 찾기`, `새로고침` 정도만 유지
- `거래 추가`는 `Add Trade` 탭이 이미 있으므로 상단 중복 노출 우선순위는 낮음

#### 3. Pinned Highlight Card

- 타임라인 시작 전 한 번만 강조
- 반드시 필요하지는 않지만, 화면의 첫인상을 카드 리스트 복붙처럼 보이지 않게 만듦

대체안:

- 오늘 가장 최근 거래
- 가장 반응 많은 거래
- 내가 아직 안 본 거래

#### 4. Timeline List

- `Feed`의 본체
- 카드 내부 구조는 최대한 반복 규칙을 유지
- 작성자 정보, 종목, 거래 타입, 핵심 수치, 코멘트, 반응이 일정한 순서여야 함

거래 카드 권장 순서:

```text
작성자 + 시간
거래 타입 칩
종목 / 종목명
수량 / 가격 / 총액 중 핵심 2~3개
코멘트 2줄
리액션 / 댓글
```

### 비워야 할 것

- 큰 상태 카드
- 장문의 피드 설명
- 너무 많은 상단 보조 CTA

### 빈 상태 와이어

```text
[Empty Illustration or Soft Panel]
아직 친구 거래가 없습니다
사람을 팔로우하면 거래가 여기 쌓입니다

[Primary]
사람 찾기

[Secondary]
내 프로필 공유하기
```

### 넓은 화면 확장안

```text
좌측 2/3
- Search / Filter Bar
- Timeline List

우측 1/3
- Pinned Highlight Card
- 추천 사용자 3명
- 내 반응이 많은 거래 요약
```

### Feed 화면 톤

- 세 화면 중 가장 `콘텐츠 중심`
- 설명보다 스크롤 경험이 먼저 살아야 함

## Portfolio Wireframe

### 화면 목적

- 내 자산 상태를 가장 깊게 읽고 관리하는 메인 분석 화면

### 핵심 질문

- 지금 내 자산은 어떻게 구성되어 있는가
- 어떤 종목이 성과를 끌고 있거나 깎고 있는가
- 장기투자 루틴은 얼마나 유지되고 있는가

### 모바일 와이어 구조

```text
[Header]
Portfolio
내 포트폴리오

[Allocation Section]
도넛 차트
상위 비중 종목
섹터 / 현금 구성

[Performance Section]
총 수익률 / 오늘 등락 / 투자원금
예상 배당
월별 흐름 또는 수익 관련 보조 차트

[Routine Section]
잔디 / 스트릭 / 투자 루틴

[Holdings List]
보유 종목 전체
- 종목명
- 비중
- 평가금액
- 손익
- 평균단가

[Management Actions]
KIS 연결
리마인더 관리
```

### 섹션별 의도

#### 1. Allocation Section

- 이 화면의 첫 분석 블록
- 자산 구성을 가장 먼저 읽게 해야 함
- 도넛과 범례는 붙이고, 섹터/현금은 하위 블록으로 배치
- 총 평가금액은 도넛 중심 레이블에서 먼저 읽히게 둔다

#### 2. Performance Section

- 총 수익률, 오늘 등락, 투자원금, 예상 배당, 최근 성과 흐름을 한곳에 모은다
- 현재 구현 중 배당 차트는 여기에 유지 가능
- 단, `Home`에서는 제거하고 이 화면으로 집중

#### 3. Routine Section

- 설계 문서에 있는 `잔디 / 스트릭 / 랭킹`을 반영하는 자리
- 아직 구현이 약한 부분이라 이번 구조안에서 명확히 자리를 확보

권장 구성:

- Heatmap
- 최근 n주 투자 기록
- 연속 기록 일수
- 이번 달 목표 달성률

#### 4. Holdings List

- 리스트 자체는 분석 이후에 와야 함
- 개별 종목 상세 진입을 위한 메인 목록
- 각 행은 동일한 정보 구조를 유지

보유 종목 행 권장 순서:

```text
종목명 / 티커
비중
평가금액
손익 + 수익률
평균단가
```

#### 5. Management Actions

- 가장 아래 관리 영역으로 분리
- 분석 화면을 액션 화면처럼 보이게 하지 않도록 톤을 낮춤
- `직접 추가 / CSV / OCR`은 여기서 반복하지 않고 하단 `+` 탭으로 보낸다

### 비워야 할 것

- `Home`에서 이미 했던 안내 문장 반복
- 화면 상단의 과도한 CTA
- 자산 구성과 무관한 소셜 정보

### 빈 상태 와이어

```text
[Hero Empty Card]
보유 종목이 아직 없습니다
첫 종목을 추가하면 자산 구성과 수익률이 정리됩니다

[Primary]
하단 + 버튼으로 첫 거래 기록

[Secondary]
KIS 연결
```

### 넓은 화면 확장안

```text
좌측 2/3
- Allocation Section
- Holdings List

우측 1/3
- Performance Section
- Routine Section
- Management Actions
```

### Portfolio 화면 톤

- 세 화면 중 가장 `분석적`이어야 함
- 밀도는 가장 높아도 되지만, 섹션 간 위계는 가장 명확해야 함

## 우선 적용 순서

1. `Home`에서 분석성 높은 블록을 줄이고 `오늘 행동` 중심으로 재구성
2. `Feed`에서 상단 상태 카드형 구조를 없애고 검색/필터 + 타임라인 우선으로 변경
3. `Portfolio`에 `루틴 / Heatmap` 영역을 추가할 자리 확보
4. 세 화면 공통으로 CTA 중복 제거

## 구현 전 체크포인트

- `Home`와 `Portfolio`가 같은 정보를 서로 얼마나 중복하는지 먼저 정리
- `Feed` 상단에서 꼭 필요한 유틸만 남기기
- `Portfolio`에 아직 없는 `루틴 시각화` 데이터를 어떤 형태로 받을지 백엔드 계약과 함께 확인
- 모바일에서 첫 화면 진입 시 각 화면 첫 스크린풀에 무엇이 보이는지 기준으로 우선순위 재점검

## 1. 우선 수정 순서

### Phase 1. 공통 레이아웃 정리

- 세 화면 모두에서 반복되는 상단 여백, 카드 간격, CTA 배치 규칙을 먼저 맞춘다.
- `DataStatusCard`가 본문 흐름을 깨지 않도록 노출 위치와 밀도를 줄인다.
- `SectionHeading`, `SurfaceCard`, `Chip`의 역할을 다시 정리해 화면별 위계를 통일한다.

이 단계에서 손대는 파일:

- `src/components/ui.tsx`
- `src/components/DataStatusCard.tsx`
- `src/theme/tokens.ts`

### Phase 2. Home 재구성

- 세 화면 중 가장 먼저 바꿔야 한다.
- 이유는 `Home`의 성격을 먼저 확정해야 `Portfolio`에서 빼야 할 정보가 명확해지기 때문이다.
- `오늘 상태 + 다음 행동 + 짧은 미리보기`만 남기고 분석 정보는 축소한다.

이 단계 목표:

- 대표 요약 카드 경량화
- Today Action 블록 추가
- Quick Actions 재배치
- 포트폴리오/피드 미리보기 축소

### Phase 3. Feed 재구성

- `Home` 이후 바로 진행한다.
- 이유는 현재 `Feed`가 가장 강하게 카드 스택형 정보 화면처럼 보이는데, 실제로는 타임라인형 화면이어야 하기 때문이다.

이 단계 목표:

- 상단 상태 카드 제거
- 검색/필터 바 도입
- 타임라인 카드 구조 표준화
- 상단 CTA 수 축소

### Phase 4. Portfolio 재구성

- 마지막으로 진행한다.
- 이유는 `Home`에서 남길 정보와 뺄 정보를 확정한 다음, `Portfolio`가 가져가야 할 분석 범위를 채워 넣는 편이 효율적이기 때문이다.

이 단계 목표:

- 분석 화면으로 역할 강화
- 자산 구성 / 성과 / 루틴 / 보유 종목 순서 확정
- `Heatmap` 포함 루틴 영역 확보

### Phase 5. 공통 QA 및 반응형 보정

- iPhone SE급 폭
- 360dp Android 폭
- 긴 종목명 / 긴 통화 문자열
- 빈 상태 / 로딩 / 오류 상태

검증 대상:

- 첫 스크린풀에서 보이는 정보 우선순위
- CTA 중복 여부
- 카드 간 시각적 위계

## 2. 공통 컴포넌트 변경 범위

원칙:

- 새 컴포넌트를 많이 만들기보다, 이미 있는 공통 부품을 조금 넓히고 필요한 2~3개만 추가한다.
- `Home`, `Feed`, `Portfolio` 3화면에서 동시에 쓰지 않는 추상화는 만들지 않는다.

### 유지

- `Page`
- `SurfaceCard`
- `PrimaryButton`
- `Chip`
- `MetricGrid`
- `Heatmap`

### 수정 필요

#### `Page`

- 현재는 모든 화면이 거의 같은 헤더 리듬을 가진다.
- 최소한 아래 옵션은 필요하다.

필요 변경:

- 헤더 아래 간격 조절 옵션
- 상단 보조 액션 배치 안정화
- 화면별 콘텐츠 간격 조절 옵션

예상 파일:

- `src/components/ui.tsx`

#### `SurfaceCard`

- 현재 `default / hero / muted` 정도만 있어 화면 역할 차이를 만들기 어렵다.

필요 변경:

- `hero` 카드의 강조 강도 정리
- 내부 패딩 크기 선택 옵션
- 필요 시 `borderless` 또는 `soft` 성격 추가

예상 파일:

- `src/components/ui.tsx`

#### `SectionHeading`

- 현재는 제목과 설명만 있다.
- 각 섹션 우측의 `전체 보기`, `더보기`, `관리` 같은 텍스트 액션을 받는 형태가 있으면 좋다.

예상 파일:

- `src/components/ui.tsx`

#### `Chip`

- 현재는 상태 표현과 필터 표현이 동일한 무게다.
- `Feed`의 필터 칩과 `Profile`의 상태 칩은 다른 성격이므로 구분이 필요하다.

필요 변경:

- 필터형 밀도 옵션
- active 상태 대비 조정
- 줄바꿈 상황에서 높이 안정화

예상 파일:

- `src/components/ui.tsx`

#### `DataStatusCard`

- 현재는 카드 한 장을 별도로 차지해서 본문 흐름을 자주 끊는다.

필요 변경:

- compact 모드
- inline 메시지 모드
- 화면 첫 진입시에만 강조하고 이후는 더 얇게 보이는 전략 검토

예상 파일:

- `src/components/DataStatusCard.tsx`

#### `Heatmap`

- 컴포넌트는 이미 있으나 `Portfolio`용 시각화로는 아직 단순하다.

필요 변경:

- 월/주 라벨
- compact card 안에 들어가는 크기 조정
- 루틴 요약 수치와 함께 배치 가능한 래퍼 필요

예상 파일:

- `src/components/Heatmap.tsx`

### 새로 추가해도 되는 공통 컴포넌트

#### `QuickActionGrid`

- `Home`의 빠른 행동 버튼 묶음용
- 2열 또는 3열 작은 액션 카드

#### `FilterBar`

- `Feed`의 검색/필터 조합용
- 입력창 + 칩 묶음

#### `PreviewList`

- `Home`에서 `친구 활동 미리보기`, `포트폴리오 미리보기`를 같은 규칙으로 표현할 때 유용
- 단, 너무 일반화하지 말고 얇은 래퍼 수준으로 유지

### 제거 또는 축소

- 섹터 / 현금 상세 바
- 긴 리마인더 목록

- `Friend Activity Preview` 2건

### 재배치

현재:

- 대형 요약 카드
- 바로 실행
- 리마인더
- 내 최근 거래
- 친구 피드

변경 후:

- 대형 요약 카드
- Today Action Strip
- Quick Actions
- Portfolio Snapshot
- Friend Activity Preview
- Routine Preview

### 구현 포인트

- `Portfolio`로 보낼 정보는 미리보기 수준으로만 남김
- 같은 CTA가 아래 카드에서 반복되지 않도록 정리

## Feed 실제 수정

### 제거 또는 축소

- `피드 상태` 큰 카드
- 상단의 과도한 2차 CTA
- `거래 추가로 이동` 버튼의 상단 중복 노출

현재 기준 참고:

- `src/screens/FeedScreen.tsx`

### 추가

- 검색 입력
- 필터 칩
- `사람 찾기 / 새로고침` 정도의 얇은 유틸 줄
- 선택적으로 `Pinned Highlight Card`

### 재배치

현재:

- 상태 카드
- 빈 상태 또는 거래 카드 목록

변경 후:

- 검색/필터 바
- 유틸 줄
- 선택적 하이라이트 카드
- 타임라인 목록
- 빈 상태 CTA

### 거래 카드 수정

- 작성자 영역과 거래 요약 영역을 더 강하게 분리
- 종목, 거래 타입, 핵심 수치, 코멘트, 리액션 순서 고정
- 코멘트는 2줄 기준으로 자르고 상세 진입에서 확장

### 구현 포인트

- `Feed`는 카드 정보판이 아니라 스크롤 콘텐츠 화면처럼 보여야 함
- 상단 설명보다 리스트 시작 속도가 중요

## Portfolio 실제 수정

### 유지

- 큰 요약 카드
- 자산 구성 블록
- 보유 종목 리스트

현재 기준 참고:

- `src/screens/PortfolioScreen.tsx`

### 추가

- `Performance Section`
- `Routine Section`
- `Heatmap + streak + monthly summary`

### 재배치

현재:

- 요약 카드
- 자산 구성
- 보유 종목
- 관리 액션

변경 후:

- 요약 카드
- 자산 구성
- 성과 블록
- 루틴 블록
- 보유 종목
- 관리 액션

### 루틴 블록 제안

- 투자 기록 Heatmap
- 최근 30일 기록 수
- 연속 기록 일수
- 이번 달 목표 달성률

### 보유 종목 리스트 수정

- 각 행의 정보 순서를 고정
- `평가금액`, `손익`, `평균단가`, `비중`의 우선순위를 다시 정리
- 모바일에서는 2행 구조, 넓은 화면에서는 1행 압축 구조 가능

### 구현 포인트

- `Portfolio`는 세 화면 중 유일하게 높은 정보 밀도를 허용
- 대신 섹션 위계는 가장 강해야 함
- `Home`에서 빠진 분석 요소는 여기서 회수
