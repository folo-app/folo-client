# YC CEO + gstack UI/UX 리팩터 마감 정리

## 목적

- `Home`, `Feed`, `Portfolio`를 서로 다른 역할로 분리한 리팩터를 리뷰와 머지 가능한 상태로 정리한다.
- 변경 범위, 화면별 핵심 변화, 체크 결과, 다음 추천 작업을 한 문서에서 확인하게 한다.

## 이번 리팩터 범위

### 공통

- `src/components/ui.tsx`
  - `utility` 톤을 추가해 hero, 본문, 지원 UI의 위계를 분리했다.
- `src/components/DataStatusCard.tsx`
  - 상태 UI를 본문보다 뒤로 물리는 지원 레이어로 정리했다.
- `src/components/Heatmap.tsx`
  - `Portfolio` 루틴용 GitHub 잔디 스타일 히트맵으로 확장했다.

### Home

- `src/screens/HomeScreen.tsx`
  - 상단을 `오늘 상태 + 오늘 할 일` 중심 대시보드로 재구성했다.
  - `빠른 확인`, `포트폴리오 미리보기`, `친구 활동 미리보기`는 메인 행동을 보조하는 요약 블록으로 축소했다.
  - 일부 데이터 실패 시 `빈 상태`처럼 보이지 않게 상태 문구와 플레이스홀더를 보정했다.

### Feed

- `src/screens/FeedScreen.tsx`
  - 상단을 `검색 -> 필터 -> 얇은 유틸 줄` 구조로 정리했다.
  - 모바일에서 하이라이트 카드와 거래 카드 밀도를 줄여 타임라인이 더 빨리 시작되게 만들었다.
  - `필터 결과 없음`과 `피드 응답 실패`를 다른 상태로 분리했다.

### Portfolio

- `src/screens/PortfolioScreen.tsx`
  - 상단을 `성과 핵심 -> 자산 구성 -> 성과 디테일 -> 루틴 -> 보유 종목` 순서로 고정했다.
  - 루틴 영역에 GitHub 잔디 스타일 거래 히트맵, streak, 목표 달성률을 추가했다.
  - 보유 종목 행을 `종목명 / 비중 / 평가금액 / 손익 / 평균단가` 비교 구조로 재정렬했다.
  - 관리 작업은 분석 본문과 경쟁하지 않는 유틸 리스트로 낮췄다.
  - 빈 상태와 에러 상태가 동시에 보이지 않게 정리했다.

### 기타

- `.gitignore`
  - `.agents/`를 ignore 처리했다.

## 화면별 핵심 한 줄

- `Home`: 오늘 무엇을 확인하고 어디로 들어가야 하는지 먼저 말하는 행동 중심 대시보드.
- `Feed`: 검색과 필터 후 바로 거래 흐름을 읽고 반응하는 타임라인 화면.
- `Portfolio`: 성과, 구성, 루틴, 보유 종목을 깊게 읽는 분석 운영 화면.
